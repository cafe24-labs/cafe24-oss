# AI SPACE 프로젝트 상태 확인

AI SPACE에 배포된 프로젝트의 상태, 로그, 목록을 조회합니다.

## 조회 절차

### 프로젝트 목록 조회

1. `list_my_projects`를 호출하여 전체 프로젝트 목록을 조회합니다
2. 각 프로젝트의 이름, URL, 상태, 런타임 정보를 표 형태로 안내합니다

### 개별 프로젝트 상태 조회

1. 사용자가 프로젝트를 지정한 경우 해당 프로젝트 사용
2. 프로젝트를 지정하지 않은 경우:
   - 프로젝트 1개: 자동 선택
   - 프로젝트 2개 이상: `list_my_projects`로 목록을 보여주고 사용자에게 선택 요청
3. `get_project_status`로 상세 상태 조회

### 표시 정보

| 항목 | 설명 |
|------|------|
| 프로젝트 URL | 배포된 사이트 접속 주소 |
| 실행 상태 | success, building, failed 등 |
| 런타임 | PHP, Node.js, Python, Static |
| 배포 모드 | single, pod |
| 리소스 사용량 | CPU, 메모리 |

### 로그 조회

사용자가 로그를 요청하면 `get_project_logs`를 호출합니다.

- 빌드 로그와 실행 로그를 구분하여 안내
- 에러가 발견되면 원인 분석과 해결 방안을 함께 제공
- "왜 에러가 나지?", "배포 실패 원인 알려줘" 등의 요청에도 로그를 조회하여 분석

### 빈 공간 확인

"빈 공간 있어?", "공간 현황" 요청 시 `list_my_spaces`를 호출하여 전체 공간 목록과 사용 가능한 빈 공간 수를 안내합니다.

## 작업 진행 상태

배포/업데이트 작업이 진행 중일 때 `get_job_status`로 진행률을 확인할 수 있습니다.

## 사이트 접속 검증

"사이트 접속 되는지 확인해줘" 요청 시 `site_verify`를 호출하여 HTTP 상태, 응답 시간, 페이지 내용을 검증합니다.

## 인증 에러 대응

MCP 도구 호출 시 `requires re-authorization`, `token expired`, `Needs authentication` 에러가 발생하면:

1. **`/login`은 해결 방법이 아닙니다.** `/login`은 Claude Code 자체 인증이며 MCP 서버 인증과 무관합니다.
2. 사용자에게 **`/mcp` 명령을 입력**하도록 안내합니다.
3. 서버 목록에서 **cafe24-aispace**를 선택하고 **Authenticate**를 진행하면 브라우저에서 Cafe24 로그인이 열립니다.
4. 로그인 완료 후 다시 요청하면 정상 동작합니다.

> 이 인증은 MCP 서버의 OAuth 토큰 갱신이며, `/login`이나 `/cafe24-login`과는 별개입니다.
